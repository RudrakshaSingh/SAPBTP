# Problem Statement 5 — Hospital OPD Appointment and Billing Analytics

| | |
|---|---|
| **Domain** | Healthcare / Hospital Management |
| **Topic** | SQL — Schema design, multi-table joins, Views, Business analytics |
| **Deliverable** | Schema `HOSPITAL_OPD_DB` + views `V_OPD_APPOINTMENT_ANALYTICS` and `V_DEPARTMENT_DAILY_REVENUE` |

---

## Business Scenario

A hospital wants to manage and analyze its OPD appointments, doctor consultations, patient visits, departments, and billing information.

Currently the hospital stores patient, doctor, department, appointment, and billing data separately. The administration wants a **consolidated reporting view** to understand:

- Which patients visited which doctor
- Which department handled the consultation
- What consultation fee was charged
- What was the payment status
- Which appointments are completed, cancelled, or pending
- Department-wise and doctor-wise revenue analysis

## Main Objective

Create a new database/schema named:

```
HOSPITAL_OPD_DB
```

Inside it, create the required tables and then create a view named:

```
V_OPD_APPOINTMENT_ANALYTICS
```

The view should provide a complete OPD appointment and billing summary.

---

## Required Tables

### 1. `PATIENTS`

Stores basic patient information.

| Required Information |
|---|
| Patient ID |
| Patient Name |
| Gender |
| Age |
| City |
| Mobile Number |

**Purpose:** identify which patient visited the hospital and from which location.

---

### 2. `DEPARTMENTS`

Stores hospital department details.

| Required Information |
|---|
| Department ID |
| Department Name |
| Floor Number |

**Purpose:** categorize doctors and appointments department-wise (Cardiology, Orthopedics, General Medicine, Pediatrics, etc.).

---

### 3. `DOCTORS`

Stores doctor details.

| Required Information |
|---|
| Doctor ID |
| Doctor Name |
| Department ID |
| Specialization |
| Consultation Fee |

**Purpose:** identify which doctor handled the consultation and what fee is charged for the visit.

---

### 4. `APPOINTMENTS`

Stores appointment booking details.

| Required Information |
|---|
| Appointment ID |
| Patient ID |
| Doctor ID |
| Appointment Date |
| Appointment Time |
| Appointment Status |

**Appointment Status** must be one of: `Completed`, `Pending`, `Cancelled`.

**Purpose:** track patient appointments with doctors.

---

### 5. `BILLING`

Stores billing and payment details.

| Required Information |
|---|
| Bill ID |
| Appointment ID |
| Bill Amount |
| Payment Mode |
| Payment Status |

**Payment Mode** must be one of: `Cash`, `UPI`, `Card`, `Insurance`.
**Payment Status** must be one of: `Paid`, `Unpaid`, `Refunded`.

**Purpose:** analyze hospital OPD revenue and payment status.

---

## Relationship Between Tables

1. One patient can have multiple appointments.
2. One doctor can handle multiple appointments.
3. One department can have multiple doctors.
4. One appointment can have one billing record.
5. Appointment data should connect patients and doctors.
6. Billing data should connect with appointment data.

---

## Main View Requirement — `V_OPD_APPOINTMENT_ANALYTICS`

Combine data from all related tables into a consolidated report with these fields:

| Field Name | Description |
|---|---|
| Appointment ID | Unique appointment number |
| Appointment Date | Date of appointment |
| Appointment Time | Time of appointment |
| Patient Name | Name of the patient |
| Patient City | City of the patient |
| Doctor Name | Name of the doctor |
| Department Name | Hospital department name |
| Specialization | Doctor specialization |
| Consultation Fee | Doctor consultation fee |
| Bill Amount | Final billed amount |
| Payment Mode | Cash, UPI, Card, or Insurance |
| Payment Status | Paid, Unpaid, or Refunded |
| Appointment Status | Completed, Pending, or Cancelled |

---

## Business Questions to Answer Using the View

### Question 1 — Completed appointments

Find all completed OPD appointments. Output: Appointment ID, Patient Name, Doctor Name, Department Name, Appointment Date, Bill Amount, Payment Status.

### Question 2 — Total revenue

Find total OPD revenue generated by the hospital. Only appointments with `Paid` payment status should be considered.

### Question 3 — Department-wise revenue

Output: Department Name, Total Revenue.

### Question 4 — Doctor-wise consultation revenue

Output: Doctor Name, Department Name, Total Appointments, Total Revenue.

### Question 5 — Pending appointments

Output: Appointment ID, Patient Name, Doctor Name, Department Name, Appointment Date, Appointment Time.

### Question 6 — Unpaid bills

Output: Patient Name, Doctor Name, Department Name, Bill Amount, Payment Status.

### Question 7 — City-wise patient visits

Output: Patient City, Total Number of Appointments.

---

## Final Assignment — `V_DEPARTMENT_DAILY_REVENUE`

Create a second view that summarizes daily OPD revenue department-wise.

| Field Name | Description |
|---|---|
| Appointment Date | Date of appointment |
| Department Name | Hospital department |
| Total Appointments | Number of completed appointments |
| Total Paid Bills | Number of paid bills |
| Total Revenue | Revenue generated |

The view should only consider:

- **Completed** appointments
- **Paid** bills
